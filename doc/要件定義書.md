要件定義書: AI駆動型モバイル動画生成・編集ツール (仮称: AIVideoCreator)1. プロジェクト概要1.1 目的スマートフォンからテキスト（台本）や画像を送信するだけで、AI生成による高品質な動画と音声を自動生成。サーバーサイドでシームレスに1本の動画へ結合し、スマホ上でミリ秒単位の微調整ができるモバイルファーストの動画制作ツール。1.2 開発アプローチ（対AI最適化戦略）Claude Code（AIエージェント）による自律開発の打率を最大化するため、コードの型が堅牢で、かつ動画編集のエコシステムが最も成熟している「TypeScript + Web技術（Capacitor + React）」を採用。重い動画結合処理はスマホ端末ではなく、サーバーサイド（Remotion）で安全に実行する。2. システム構成・技術スタックレイヤー採用技術選定理由共通言語TypeScriptClaude Codeによるバグの自己発見・修正精度を最大化するため。フロント（アプリ）React + Tailwind CSS(Capacitorによるネイティブ化)タイムラインUIや細かな調整画面を、Webの豊富なコンポーネント資産を活用して最速で実装するため。バックエンド / APINext.js (App Router)ディレクトリ構造が明快でAIが迷子にならない。動画生成APIとの中継。動画・音声結合Remotion (Node.js環境)動画のタイムライン、結合、音声音量のクロスフェードを**「TypeScriptのコード」として記述・レンダリング**できる唯一無二のフレームワーク。インフラ / データベースSupabase (Auth, DB, Storage)ユーザー管理と、生成された重い動画/音声ファイルのストレージ管理を一体化。外部AI APIRunWay / HeyGen / ElevenLabs など動画生成、アバター生成、高精度音声合成。3. 機能要件3.1 台本ベース動画・音声自動生成機能台本（スクリプト）の自動解析: ユーザーがスマホから入力した長文の台本を、AIが「シーン（章）」ごとに自動分割。マルチモーダル生成: 分割されたシーンごとに、AI動画生成APIとAI音声（ナレーション）生成APIを非同期で同時キック。進捗通知: 生成に時間がかかるため、バックグラウンド処理（Queue）とし、完了時はスマホにPush通知。3.2 サーバーサイド・シームレス結合機能（Remotion駆動）映像のシームレス連結: AIが生成したぶつ切りの動画クリップを、Remotion上でディゾルブ（クロスフェード）等のトランジションを挟んで滑らかに1本化。音声のミリ秒単位マージ: ナレーション音声のつなぎ目が不自然（無音の空白）にならないよう、波形レベルで適切なバッファ（無音区間）を計算して結合。BGMとナレーションのダッキング: ナレーションが流れている間、BGMの音量を自動で下げる（ダッキング処理）。3.3 スマホUIによる微調整・編集機能タイムラインプレビュー: 結合された動画をスマホ（9:16縦型）でストリーミング再生。シーン順序入れ替え・トリミング: ドラッグ＆ドロップによるシーンの並び替え、およびミリ秒単位の長さ調整。テキスト（字幕）オーバーレイ修正: AIが生成したナレーションに合わせた自動字幕の、誤字脱字をスマホ上で直接修正。部分再生成（リテイク）: 「シーン3の動画だけ別のプロンプトで作り直す」ことができる機能。4. 非機能要件端末負荷の最小化: 重いエンコード（mp4出力）処理はすべてNext.js/Remotionサーバー側で行い、スマホ端末のバッテリー消費や発熱を防ぐ。セキュリティ: ユーザーがアップロードした画像や生成された動画は、Supabase StorageのRow Level Security (RLS) により厳格に保護。
