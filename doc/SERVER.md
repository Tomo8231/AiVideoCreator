# PCを生成サーバーとして使う（スマホから投入）

このPCを「動画生成サーバー」として常時起動し、**スマホのブラウザから台本＋画像を
送信 → サーバー側で生成 → スマホで進捗確認**する使い方の手順です。

```
スマホ(ブラウザ)
   │  台本＋画像を送信 (POST /api/jobs)
   ▼
PCサーバー (Next.js, このリポジトリ)
   ├─ ジョブを登録して即 id を返す（非同期）
   ├─ 台本をシーン分割
   ├─ シーンごとに 画像/動画(ComfyUI or RunWay) ＋ 音声(ElevenLabs) を生成
   └─ 進捗をジョブに反映（スマホが GET /api/jobs/{id} でポーリング）
```

生成は PC 側で進むので、**スマホを閉じても継続**します。あとから履歴
（`/jobs`）で状況・成果物を確認できます。

## 1. サーバー側（PC）の準備

### キーやプロバイダを `.env` に設定
スマホはキーを送りません。生成に使うキー類は**サーバーの `.env`** に置きます。

```bash
# 例: ローカル ComfyUI で動画(画像)生成 + ElevenLabs で音声
COMFYUI_BASE_URL=http://127.0.0.1:8188
COMFYUI_WORKFLOW_PATH=./image_flux2.json   # API形式ワークフロー(プレースホルダ入り)
ELEVENLABS_API_KEY=...                       # 任意（無ければモック音声）
ANTHROPIC_API_KEY=...                        # 任意（無ければヒューリスティック分割）
```

- ComfyUI を使う場合は **ComfyUI を先に起動**しておく（既定 `127.0.0.1:8188`）。
- ワークフローJSONはプレースホルダ `%PROMPT%` / `%IMAGE%` / `%SEED%` を埋めておく
  （詳細は設定画面の「キーの取得方法」→ ComfyUI 節、または `image_flux2.json` を参照）。
- いずれのキーも未設定ならモックで動作します（疎通確認に便利）。

### サーバー起動
```bat
quickstart.bat prod          :: 本番ビルドして起動（推奨）
quickstart.bat prod 3000     :: ポート指定
```
`npm run dev` / `npm start` は `-H 0.0.0.0` 付きで全インターフェースにバインドします。

## 2. スマホからアクセスする

### A. インターネット越し（トンネル）
外出先のスマホからも使えます。いずれかでPCのサーバーを公開します。

- **Cloudflare Tunnel**（無料・おすすめ）
  ```bash
  cloudflared tunnel --url http://localhost:3000
  ```
  表示される `https://xxxx.trycloudflare.com` をスマホで開く。
- **ngrok**
  ```bash
  ngrok http 3000
  ```
  表示される `https://xxxx.ngrok-free.app` をスマホで開く。
- **Tailscale**（VPN）: PCとスマホを同じ tailnet に入れ、PCの Tailscale IP:3000 を開く。

アプリは同一オリジンで `/api` を呼ぶので、トンネルURLで開けばそのまま動きます
（追加のCORS設定は不要）。

### B. 同じ Wi-Fi（LAN）
PCの LAN IP を調べてスマホで開きます。
```powershell
ipconfig   # IPv4 アドレス（例 192.168.1.23）を確認
```
スマホで `http://192.168.1.23:3000` を開く。
（Windows ファイアウォールでポート 3000 の受信許可が必要な場合があります）

## 3. 使い方
1. スマホでサーバーのURLを開く。
2. タイトル・台本・（任意で）起点画像を入力して「サーバーで生成する」。
3. 状況画面（`/jobs/{id}`）に移動。進捗バーとシーン別の映像/音声ステータスが
   ポーリングで更新されます。完了後は各シーンの成果物リンクから確認できます。
4. 履歴は右上のアイコン（`/jobs`）から。

## 注意
- ジョブはサーバープロセスのメモリ＋`data/jobs.json` に保持されます
  （`data/` は Git 管理外）。サーバーを動かし続けてください。
- ComfyUI の生成物URLは ComfyUI サーバー（`127.0.0.1:8188`）を指します。
  トンネル経由のスマホから直接その画像を開くにはトンネル側の設定が別途必要です
  （状況確認・進捗はトンネルだけで利用できます）。
- この構成は**常時起動の Node サーバー前提**です（Vercel 等のサーバーレスでは
  バックグラウンド継続生成ができません）。
