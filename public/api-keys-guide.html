APIキーの取得方法

AIVideoCreator で実際のAI生成・クラウド保存を使うための設定ガイド

キーは未設定でもアプリは動きます。 その場合は生成がモック（ダミー）になり、保存はこの端末内（localStorage）になります。 本物のAIで生成したいキーだけ設定すればOKです。
まず無料で全機能（台本分割・タイムライン編集・字幕・結合）を試すなら、アプリの 設定 → 生成モード → モック を選んでください。
実APIの多くはクレジット購入や有料プランが前提です（RunWay はクレジット、 ElevenLabs の API 音声は有料プランが必要なことがあります）。各節の「よくあるエラー」を参照。

① キーの設定方法（2通り）

取得したキーは、次のどちらかの方法で設定します。

方法手順向いている用途
設定画面 アプリ右上の歯車 → 「APIキー」に貼り付け。ブラウザに保存され、生成時にサーバーへ送信。 手軽に試す。自分専用の端末。
.env ファイル プロジェクト直下の .env にキーを記入してサーバーを再起動。 本番・共有環境（Vercel の環境変数も同じ考え方）。

※ 設定できる項目の一覧は .env.example にもまとまっています。

② ElevenLabs 任意

シーンごとのナレーション音声を合成します。未設定ならモック音声（尺だけ推定）。

取得手順

  1. elevenlabs.io で登録・ログイン。
  2. 右上のアカウントメニュー → API Keys（または Profile + API Key）。
  3. Create API Key でキーを発行し、コピー。
  4. （任意）使いたい声の Voice ID を Voices 画面で確認。

設定する値

  • ELEVENLABS_API_KEY … 発行したキー
  • ELEVENLABS_VOICE_ID … 任意（未設定はデフォルトの声）

よくあるエラー

  • 402: paid_plan_required / Free users cannot use library voices … 無料プランは API からライブラリ音声（Rachel 等のプリメイド音声）を使えません。 次のいずれかで解決します:
    • 有料プラン（Starter 以上）にアップグレードする。
    • 自分のアカウントの音声（My Voices / 自作・クローン音声）の Voice ID を ELEVENLABS_VOICE_ID に設定する。
  • 401: invalid_api_key … キーの打ち間違い・失効。再発行して設定し直す。

③ RunWay 任意

シーンの動画クリップを生成します（RunwayML Developer API）。未設定ならモック動画。

RunWay の動画は 「起点画像」から動かす（image_to_video）方式です。 エディタの各シーンで「起点画像」を添付するか、全シーン共通の既定画像 RUNWAY_DEFAULT_IMAGE を設定してください。画像が無いと生成はエラーになります。

取得手順

  1. dev.runwayml.com（Developer ポータル）にログイン。
  2. API Keys で新しいキーを発行し、コピー。
  3. 従量課金のためクレジット残高の登録が必要な場合があります。

設定する値

  • RUNWAY_API_KEY … 発行したキー
  • RUNWAY_DEFAULT_IMAGE … 任意。起点画像のURL（全シーン共通の既定）
  • RUNWAY_POLL_INTERVAL_MS / RUNWAY_POLL_TIMEOUT_MS … 任意。ポーリング間隔・上限

よくあるエラー

  • 400: not enough credits（クレジット不足）… RunWay は従量課金です。 ダッシュボードの Billing / Credits でクレジットを購入してください。 無料枠だけでは API からの動画生成はできないことがあります。
  • 起点画像が必要です … シーンに画像を添付するか RUNWAY_DEFAULT_IMAGE を設定してください。

③-2 ComfyUI（ローカル動画生成） 任意・無料

自分のPCで動く ComfyUI を使うと、課金なしで動画を生成できます。 RunWay の代わりにローカルで動かす選択肢です。

ローカルの ComfyUI に接続するため、このアプリも同じPCで起動してください （サーバー側から 127.0.0.1:8188 へ接続します）。Vercel 等の外部にデプロイ した場合は接続できません。

使い方

  1. ComfyUI を起動（既定 http://127.0.0.1:8188）。動画生成系のワークフロー （SVD / WAN / LTX / AnimateDiff など、導入済みのモデルに合わせて）を用意。
  2. ComfyUI のメニュー Save (API Format) でワークフローJSONを書き出す。
  3. JSON 内の差し込みたい箇所を、次のプレースホルダに置き換える:
    • %PROMPT% … プロンプト（CLIPTextEncode の text など）
    • %IMAGE% … 起点画像のファイル名（LoadImage の image など。画像は自動アップロードされます）
    • %SEED% … 乱数 seed（毎回変えたい場合）
  4. アプリの 設定 → 動画生成プロバイダ → ComfyUI を選び、URLと ワークフローJSONを貼り付けて保存。

設定する値（env で常用する場合）

  • COMFYUI_BASE_URL … 例 http://127.0.0.1:8188
  • COMFYUI_WORKFLOW … API形式JSON（長い場合は設定画面推奨）
  • COMFYUI_POLL_INTERVAL_MS / COMFYUI_POLL_TIMEOUT_MS … 任意

※ ワークフローはモデル/ノード構成ごとに異なります。お使いの環境で 単体で動くワークフローを書き出して使ってください。

④ Anthropic 任意

台本を Claude で意味的にシーン分割します。未設定なら簡易（ヒューリスティック）分割。

取得手順

  1. console.anthropic.com にログイン。
  2. 左メニュー API Keys → Create Key で発行し、コピー。
  3. Billing でクレジットの登録が必要な場合があります。

設定する値

  • ANTHROPIC_API_KEY … 発行したキー

⑤ Supabase 任意

ログインとプロジェクトのクラウド保存・起点画像の保存に使います。未設定なら端末内保存のまま。

取得手順

  1. supabase.com でプロジェクトを作成。
  2. SQL Editor を開き、リポジトリ同梱の supabase/schema.sql の内容を貼り付けて Run。 （projects テーブル・media バケット・RLS が作成されます）
  3. Project Settings → API で次の2つをコピー:
    • Project URL
    • Project API keys の anon public キー

設定する値（.env に記入）

  • NEXT_PUBLIC_SUPABASE_URL … Project URL
  • NEXT_PUBLIC_SUPABASE_ANON_KEY … anon public キー

※ これらは NEXT_PUBLIC_ 接頭辞のため、設定後はサーバーの再起動（再ビルド）が必要です。 設定画面ではなく .env で設定してください。

設定後、アプリの「設定 → アカウント」からログインし、/projects で保存・読み込みができます。 RLS により各ユーザーは自分のデータ（projects 行と media/{userId}/ フォルダ）のみアクセスできます。

⑥ セキュリティの注意

  • APIキーは秘密情報です。スクリーンショットやチャットで共有しないでください。
  • 設定画面に入れたキーはブラウザ（localStorage）に保存されます。共有端末では使わないでください。
  • より安全に運用するなら、キーは .env（またはホスティングの環境変数）に置き、設定画面は空のままにします。
  • キーが漏れた疑いがあるときは、各サービスの管理画面で失効（Revoke）して再発行してください。
